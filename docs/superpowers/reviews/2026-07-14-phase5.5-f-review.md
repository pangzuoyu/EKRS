## Findings

### [P2] (confidence: 9/10) `ekrs_rag/observability/audit.py:43-44` — `namer`/`rotator` set externally instead of inside `RebuildingRotatingFileHandler`

```python
handler = RebuildingRotatingFileHandler(
    str(path), maxBytes=..., backupCount=..., encoding="utf-8",
    on_rollover=on_rollover,
)
handler.namer = gzip_namer        # ← external config
handler.rotator = gzip_rotator    # ← external config
```

`RebuildingRotatingFileHandler` advertises "RotatingFileHandler that invokes on_rollover after each rotation" but in practice needs the caller to also wire namer/rotator. Self-contained class would set them as defaults:

```python
# ekrs_rag/observability/audit_handler.py
class RebuildingRotatingFileHandler(RotatingFileHandler):
    def __init__(self, *args, on_rollover=None, namer=gzip_namer, rotator=gzip_rotator, **kwargs):
        super().__init__(*args, **kwargs)
        self.namer = namer
        self.rotator = rotator
        self._on_rollover = on_rollover
```

Then AuditWriter passes only `on_rollover`. Less surprise, fewer lines.

**Fix cost**: ~10 LOC. Optional override `namer=None/rotator=None` keeps the API open for tests.

### [P2] (confidence: 8/10) Missing integration test — rollover → AuditIndex rebuild callback

The wiring `on_rollover=_on_audit_rollover` lives only in `main.py:210`. Neither `_on_audit_rollover` (the closure) nor its dispatch into `_audit_index.build()` is exercised by a test. Failure modes that go undetected:

1. Closure references `_audit_index` global too early (it's set AFTER AuditWriter at line 211 in main.py, but rollover only fires after 100MB, so lazy read is correct — but a test would prove it).
2. `_audit_index.build()` raises during rotation, request thread dies (today: caught + logged in `doRollover`'s wrapper, but the wrapper itself is in audit_handler.py and untested for this propagation).

**Fix cost**: 1 new integration test, ~30 LOC. Spawn an AuditWriter with a real AuditIndex attached, force rotation, assert index was rebuilt.

### [P3] (confidence: 7/10) AuditWriter.write uses lazy import for `get_skip_audit`

```python
def write(self, event_type: str, **kwargs) -> bool:
    from ekrs_rag.observability.trace import get_skip_audit
    ...
```

Lazy import is consistent with the existing pattern in `observability.py:44` (`get_writer`), so it's not wrong — but it IS evaluated on EVERY audit write, even though `trace.py` is imported by `audit_handler.py` transitively (audit_handler → audit → trace would form a cycle otherwise). Actually checking: trace.py has no audit import, so no cycle. The lazy import is gratuitous.

**Fix cost**: 1 line hoisted to module top.

### [P3] (confidence: 5/10) No test for `_current_offset()` defensive branches

`_current_offset()` returns 0 when `h.stream.closed` or on OSError. These branches exist to keep the index from registering garbage offsets after handler shutdown. They're untested. Probably fine in practice — `audit.log` is written by app state, shutdown closes the handler — but a smoke test costs 5 LOC.
