# Phase 5.5 E: Routes 全局 → FastAPI `Depends`

**日期**: 2026-07-13
**范围**: 5 个模块级全局 → FastAPI `Depends(get_x)` + `app.state.x` + `app.dependency_overrides` 测试模式
**非目标**: A/B/C/D(Phase 5.5 其他子系统)、Iron Rules、Audit schema、Phase 5 replay 行为

---

## 目标

替换 `routes/constraints.py` 和 `routes/ingestion.py` 中的 5 个模块级 `_X` 全局为标准 FastAPI 依赖注入。原因:

1. 模块级单例绕过 FastAPI 生命周期,test 间状态污染靠 `_reset_module_globals` fixture 修补(Task 17 I4 临时方案)
2. 测试 hook 与生产代码 hook 共用 `set_X()` 函数,语义模糊
3. Future SSoT:`app.state` 是 FastAPI 唯一的状态容器

---

## 现状

```
rag/ekrs_rag/main.py lifespan:
    constraints.set_retriever(_retriever)        # module global
    ingestion.set_pipeline(_pipeline)
    ingestion.set_redis_lock(_redis_lock)
    ingestion.set_task_repo(_task_repo)
    constraints.set_audit_index(_audit_index)

routes/constraints.py:
    _retriever: EKRSRetriever | None = None
    _audit_index: AuditIndex | None = None
    def set_retriever(...): global _retriever = ...
    def set_audit_index(...): global _audit_index = ...

routes/ingestion.py: 类似 3 个全局 + 3 个 setter

路由读取:  request.app.state.X or _X   (双 fallback)
测试注入:  cmod.set_X(mock)              (直接覆写全局)
```

5 个 `set_X` 函数被以下文件调用:`main.py`(生产启动)+ 3 个 integration test(测试注入)。

---

## 设计决策(已 user-approved)

| # | 决策 | 理由 |
|---|---|---|
| D1 | 改 5 个全局而非仅 `_retriever` | 与 Task 17 I4 现件同源,一并清干净 |
| D2 | 用 `app.dependency_overrides`(不是 lifespan setter + Depends 双轨) | FastAPI idiom,测试用 overrides,生产用 app.state,语义清晰 |
| D3 | 硬删除 `set_X` 函数 + `_reset_module_globals` fixture | 测试改写为 overrides,无 BC shim,避免长期双路径 |
| D4 | `get_audit_index` Optional 返回 None;其他 4 个 strict 抛 503 | AuditIndex 是 replay 专用,normal flow 不应 fail |
| D5 | Strict 依赖失败 = 503(不是 500 或 RuntimeError) | 与既有 replay 分支 503 一致,明确"未配置"语义 |

## 详细设计

### 依赖函数

| 路径 | 函数 | 严格? | 错误 (未初始化) |
|---|---|---|---|
| `routes/constraints.py` | `get_retriever(request) -> EKRSRetriever` | strict | 503 |
| `routes/constraints.py` | `get_audit_index(request) -> AuditIndex \| None` | optional | (返回 None) |
| `routes/ingestion.py` | `get_pipeline(request) -> IngestionPipeline` | strict | 503 |
| `routes/ingestion.py` | `get_redis_lock(request) -> RedisLock` | strict | 503 |
| `routes/ingestion.py` | `get_task_repo(request) -> TaskRepo` | strict | 503 |

所有依赖函数同步 `def`(非 async),仅读 `request.app.state.x`。

### 路由签名变化

```python
# /v1/constraints
async def query_constraints(
    query: ConstraintQuery,
    retriever: EKRSRetriever = Depends(get_retriever),
    audit_index: AuditIndex | None = Depends(get_audit_index),
    _auth: None = Depends(require_parser_token),
) -> ConstraintQueryResponse:
    # 删除 request: Request 形参(retriever 已注入)
    # 删除 `getattr(request.app.state, "retriever", None) or _retriever` 双 fallback
    # 重命名内部变量: retriever 不再需要属性访问
    if query.replay:
        if not query.replay_trace_id:
            raise HTTPException(400, "replay_trace_id required")
        if audit_index is None:                          # replay-only 503 检查保留
            raise HTTPException(503, "audit index not initialized")
        ...

# /v1/ingestion/notify
async def ingestion_notify(
    body: IngestionNotification,
    pipeline: IngestionPipeline = Depends(get_pipeline),
    lock: RedisLock = Depends(get_redis_lock),
    repo: TaskRepo = Depends(get_task_repo),
    _auth: None = Depends(require_parser_token),
): ...

# /v1/ingestion/status/{doc_hash}
async def ingestion_status(
    doc_hash: str,
    repo: TaskRepo = Depends(get_task_repo),
): ...

# /v1/ingestion/replay
async def ingestion_replay(
    body: IngestionReplayRequest,
    repo: TaskRepo = Depends(get_task_repo),
    _auth: None = Depends(require_parser_token),
): ...
```

### main.py 改动

```python
# 之前
constraints.set_retriever(_retriever)
ingestion.set_pipeline(_pipeline)
ingestion.set_redis_lock(_redis_lock)
ingestion.set_task_repo(_task_repo)
constraints.set_audit_index(_audit_index)

# 之后
app.state.retriever = _retriever
app.state.audit_index = _audit_index     # AuditIndex build 完成后赋值
app.state.pipeline = _pipeline
app.state.redis_lock = _redis_lock
app.state.task_repo = _task_repo
```

`import: from ekrs_rag.api.routes import constraints, ingestion` 不变,但 `constraints.set_audit_index(...)` 等 5 调用全部删除。

## 数据流

### 启动期

```
uvicorn → create_app() → lifespan() 启动
    │  创建 5 个单例
    ├─ app.state.X = X(...)
    └─ yield  # app 接受请求
```

### 请求期

```
HTTP POST /v1/constraints
    │
    ▼  FastAPI 解析依赖
get_retriever(request)      → app.state.retriever → EKRSRetriever | raise 503
get_audit_index(request)    → app.state.audit_index → AuditIndex | None
require_parser_token        → 校验 X-Parser-Token
    │
    ▼  handler 收到 3 个 kwargs
query_constraints(query, retriever, audit_index, _auth=None)
```

---

## 错误处理矩阵

| 场景 | HTTP | 来源 |
|---|---|---|
| `app.state.x` 未设置(strict 依赖) | 503 | `get_x()` raises `HTTPException(503, "X not initialized")` |
| Replay 缺少 audit_index | 503 | 路由内 `if audit_index is None: 503` |
| Replay trace_id 无记录 | 400 "no_prior_solve" | 不变 |
| Replay trace_id 记录不完整 | 400 "incomplete_prior_solve" | 不变 |
| Retriever 返回 0 chunks (Gate 1) | 404 "Insufficient recall" | 不变 |
| Missing/invalid X-Parser-Token | 403 | require_parser_token |
| Normal flow Auth disabled | no-op | PARSER_TOKEN="" 短路 |

## 备选方案(已不采纳)

### A1: 仅 `_retriever` 一个全局迁移
- 优点: 最小变更,只改 constraints.py + 2 个测试
- 缺点: 与 Task 17 I4 同源的 `_audit_index` 仍走原路,留尾巴
- 决策: 不采纳,user 选 D1"5 个全换"

### A2: 保留 `set_X` 为 deprecation wrapper(转写 app.state)
- 优点: BC 友好
- 缺点: 长期双路径,违反 KISS。外部调用者只在 main.py + 3 test,易迁移
- 决策: 不采纳,user 选 D3"硬删除 + 测试改造"

### A3: lifespan 写 module-level cache + Depends 读 cache
- 优点: 单例语义保留
- 缺点: 全局状态仍可见,违反 FastAPI idiomatic state 模式
- 决策: 不采纳,user 选 D2 "app.dependency_overrides"
