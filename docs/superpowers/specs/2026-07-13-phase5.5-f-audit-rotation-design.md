# Phase 5.5 F: Audit Log Rotation + /healthz Filter

## Goal

Solve the audit.log bloat problem (1377 MB / 61122 events observed) by:
1. Bounding audit.log size via RotatingFileHandler (100 MB × 5, gzip)
2. Suppressing audit writes for `/healthz` (64.8% of event volume)

Without losing replay ability or violating the Iron Rules.

## Background

### Current state (post-Phase 5.5 E)

**audit.py** uses `logging.FileHandler` — file grows unbounded. CLAUDE.md says
> Audit log (`audit.log`): permanent, never disabled, records every solve with evidence

**observability/middleware/observability.py** emits `endpoint_started`/`endpoint_completed` for every request including `/healthz` (called by k8s liveness probe every few seconds).

### Trigger

`audit.log = 1377 MB / 61122 events`; endpoint_started/completed = 64.8% volume.
After /healthz filter + rotation: estimate ~50 MB current file + 5 × 100 MB gzip = 550 MB max.

### User decisions (locked)

- RotatingFileHandler, 100 MB × 5, **gzip** rotated files
- Skip ALL audit events for /healthz (not just lifecycle events)
- AuditIndex rebuild on rotation: scan only the current file (skip rotated .gz files)
- CLAUDE.md: rewrite "permanent, never disabled" → "permanent, size-bounded by rotation"

## Non-goals

- Compressing existing audit.log (rotation only affects new writes)
- Backing up rotated files to object storage
- Changing AuditIndex.build() algorithm (still linear scan)
- Multi-process coordination (current process owns the writer; Phase 5.5 D pattern)
- Pre-Phase-5 events that may exist in old audit.log (handled by current build())

## Architecture

### 1. RotatingFileHandler with gzip rotator

Python stdlib `logging.handlers.RotatingFileHandler` supports `namer` + `rotator`
parameters (Python 3.3+). We pass functions that:
- `namer`: append `.gz` to rotated file names (`audit.log.1` → `audit.log.1.gz`)
- `rotator`: gzip the file content during rollover

```python
# ekrs_rag/observability/audit_handler.py
import gzip, os, shutil
from logging.handlers import RotatingFileHandler


def gzip_namer(name: str) -> str:
    return name + ".gz"


def gzip_rotator(source: str, dest: str) -> None:
    with open(source, 'rb') as f_in, gzip.open(dest, 'wb') as f_out:
        shutil.copyfileobj(f_in, f_out)
    os.remove(source)


class RebuildingRotatingFileHandler(RotatingFileHandler):
    """RotatingFileHandler that calls on_rollover after each rollover.

    Used by AuditWriter so AuditIndex can rebuild offsets after the file
    rotates (old offsets point into the now-renamed audit.log.1.gz).
    """

    def __init__(self, filename, mode='a', maxBytes=0, backupCount=0,
                 encoding=None, delay=False, errors=None, on_rollover=None):
        super().__init__(filename, mode, maxBytes, backupCount,
                         encoding, delay, errors)
        self._on_rollover = on_rollover

    def doRollover(self):
        super().doRollover()
        if self._on_rollover is not None:
            try:
                self._on_rollover()
            except Exception:  # pragma: no cover
                # Audit writes already wrapped in try/except in AuditWriter.write;
                # we don't want a buggy rebuild to crash the request thread.
                import logging, traceback
                logging.getLogger("ekrs.audit.rollover").error(
                    "rollover rebuild failed: %s", traceback.format_exc()
                )
```

### 2. AuditWriter uses rotation + on_rollover callback

```python
# ekrs_rag/observability/audit.py
class AuditWriter(AuditLogger):
    def __init__(self, audit_log_path: str, on_rollover=None):
        super().__init__(name="ekrs.audit")
        path = Path(audit_log_path)
        path.parent.mkdir(parents=True, exist_ok=True)

        handler = RebuildingRotatingFileHandler(
            str(path),
            maxBytes=100 * 1024 * 1024,  # 100 MB
            backupCount=5,
            encoding="utf-8",
            namer=gzip_namer,
            rotator=gzip_rotator,
            on_rollover=on_rollover,
        )
        handler.setFormatter(logging.Formatter("%(message)s"))
        self._logger.addHandler(handler)
        self._file_handler = handler
```

No change to `_current_offset()` (still `handler.stream.tell()` — works because
`emit()` is called before each write and rollover happens inside emit()).

### 3. /healthz skip via contextvar

New contextvar in `ekrs_rag/observability/trace.py`:

```python
_skip_audit: ContextVar[bool] = ContextVar("ekrs_skip_audit", default=False)


def set_skip_audit(skip: bool) -> Token:
    return _skip_audit.set(skip)


def reset_skip_audit(token: Token) -> None:
    _skip_audit.reset(token)


def get_skip_audit() -> bool:
    return _skip_audit.get()
```

`AuditWriter.write()` checks the flag at the top:

```python
def write(self, event_type: str, **kwargs) -> bool:
    from ekrs_rag.observability.trace import get_skip_audit
    if get_skip_audit():
        return False  # silently dropped — /healthz no-audit
    try:
        ...
```

`ObservabilityMiddleware.dispatch` sets the flag for /healthz:

```python
async def dispatch(self, request: Request, call_next):
    trace_id = extract_or_generate_trace_id(dict(request.headers))
    token = set_trace_id(trace_id)
    skip_token = set_skip_audit(request.url.path == "/healthz")
    start = time.monotonic()
    try:
        ...
    finally:
        reset_skip_audit(skip_token)
        reset_trace_id(token)
```

Why contextvar (not a request-scoped dict): audit writes may happen deep in
request handlers via `get_writer()`; we want a transparent mechanism that
doesn't require passing the flag through every call. ContextVar + `finally`
reset mirrors the existing `trace_id` pattern in the same file.

### 4. main.py lifespan wires the on_rollover callback

AuditWriter is created BEFORE AuditIndex in lifespan, but the on_rollover
callback is only invoked AFTER the file exceeds 100 MB — by which time
`_audit_index` is set. Use a closure that captures `_audit_index` by name
(module global; we read it lazily inside the callback):

```python
# main.py lifespan, after AuditWriter creation:
def _rebuild_index():
    if _audit_index is not None:
        _audit_index.build()

_audit_writer = AuditWriter(audit_path, on_rollover=_rebuild_index)
```

`_audit_index.build()` clears the in-memory offset dict, then re-scans the
CURRENT audit.log (post-rollover, it's a fresh empty file that the rollover
just created). Old `.gz` files are NOT scanned (user decision).

Rebuild is synchronous (~1s for 100 MB file on SSD). It blocks the request
thread that triggered rollover. Acceptable: rollover is rare (once per ~100MB
of audit writes). For high-throughput, future optimization: schedule rebuild
on a background thread.

## Risks

1. **AuditIndex rebuild blocks request thread** — mitigatable with async/threading.
   Defer to Phase 5.5 F+ if needed.
2. **Gzip file naming collision**: if process crashes between super().doRollover
   and gzip_rotator running, we may have un-gzipped `.1` file AND fresh
   `.log` file. Mitigation: rotator is called inside doRollover() — single
   method call, single point of failure.
3. **AuditIndex offsets across rotation**: existing offsets were for a file
   that's now `audit.log.1.gz`. After rebuild, only new offsets (post-rotation)
   are kept. Old trace_ids from before rotation are NOT replayable. Per user
   decision: "只扫新 audit.log" (only scan new file). Acceptable.
4. **Concurrent rollover**: RotatingFileHandler has known issues under
   multi-process writes. Single-writer pattern (Phase 5.5 D) avoids this.
